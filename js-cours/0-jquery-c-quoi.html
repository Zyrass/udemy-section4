POURQUOI JQUERY !?

Pourquoi Jquery

Jquery permet de manipuler les éléments HTML, CSS voir même d'ajouter des animations.. et bien plus encore...

Nous pouvons par exemple, gérer la gestion des évènements (le clic, sur un hover, quand on entre avec la souris, quand on en sort etc...).

Il faut savoir aussi que quand on code on a souvent des problèmes de compatibilité avec nos navigateurs. En gros un code spécifique
ne sera pas interprété de la même manière sur tel ou tel navigateur (firefox, chrome ou bien safari etc...)
Ainsi, franchement on y gagne un temps considérable.

(Pour qu'un code soit fonctionnel sur tout les navigateurs: -webkit- pour chrome ; o- pour opera; ms- pour microsoft etc.. ;)

QUE FAIRE EN PREMIER ?

QUE FAIRE EN PREMIER ?

Avant toutes choses il faut télécharger ou bien utilisé une version uplodé en ligne de jQuery.
Plusieurs versions existe j'y reviendrais plus tard.

Ci-dessous un exemple d'utilisation de jquery

                // On charge en 1ER jquery ( - ici la version 3.1.1 minifié - )
                <script src="https://code.jquery.com/jquery-3.1.1.min.js" ></script>
                // Ensuite, on charge notre script perso sur Jquery dans un fichier externe
                <script src="myScript.js"></script>

DEUX VERSIONS D'UN MEME FICHIER, POURQUOI ?

DEUX VERSIONS D'UN MEME FICHIER, POURQUOI ?

Version Minifié (minified)

Vous trouverez tout le temps, une version minifié (minified) à disposition du grand public. Elle est compressé, dépourvu de commentaire, d'espaces et saut de ligne.

Ainsi, le fichier minifié est clairement plus légé et donc plus rapide que sa version non compréssé. Elle est utilisée pour la mise en production

Version Non-Compressé

Cette version est uniquement utilisé pour le développement. Cette version comprends les espaces, saut de ligne, commentaires etc...
Si vous avez une coquille, elle sera clairement plus simple à relire. En revanche vous perdrez du temps sur l'ouverture du fichier.

Il faut savoir qu'un code non minifié sera très rapide si quelques personnes l'ouvre mais si ces quelques personnes
se transforme en millier, l'ouverture sera multiplié part le nombre de personne cherchant à charger votre page. Ainsi donc, une fois que vous aurez fini et validé votre code, vous serez obligé de passé sur un compilateur afin de minifié ce dernier.

UTILISER JQUERY DE PLUSIEURS MANIERES

UTILISER JQUERY DE PLUSIEURS MANIERES

1 - HTML DIRECTEMENT

A la fin de notre code ( bien entendu, avant la balise de fermeture du body </body> ) afin qu'on puisse laissé le temps
au DOM se charger. Si vous souhaitez mettre le script avant, c'est à dire dans le head, le script se chargera en 1er et donc le DOM n'aura pas eu le temps de se chargé

Donc après notre code html dans le corps du document, nous ouvrirons la balise script comme ceci :

                <script>Votre code jquery ici...</script>
Enfin entre ces balises, nous devrons écrire notre code jquery. Mais avant bien entendu vous aurez ajouté votre script JQUERY

Exemple d'utilisation d'un script jquery

                        code HTML ici...

                        <script src="https://code.jquery.com/jquery-3.1.1.min.js"</script>
                        <script>
                            $(document).ready(function(){                     

                                // votre code ici...

                            });
                        </script>
                    </body>
                </html>

2 - FICHIER EXTERNE

Même chose que précédemment, pour rappel, avant </body>, il faut ouvrir une balise script, sauf que là
on y ajoute un attribut src (source) et on y ajoute notre fichier qui aura comme extension : .js.

Attention, je me répète mais surtout ne pas oublié le fichier jquery qui pour rappel doit être placé avant tout script perso.

Exemple :

Fichier HTML :

                        code HTML ici...

                        <script src="https://code.jquery.com/jquery-3.1.1.min.js"></script>
                        <script src="myScript.js"></script>
                    </body>
                </html>
myScript.js : ( Contenu du Script externe )
                $(document).ready(function(){                                              
                    // votre code ici...
                });

3 - CDN (Content Délivery Network)

Un CDN est un regroupement de plusieurs version d'un framework ou fichier quelconque hébergé sur un serveur distant.
l'avantage d'un fichier CDN c'est qu'il n'ai aucunement nécéssaire d'avoir ces fichiers localement.

En revanche, l'inconvéniant et oui il y en a un, c'est qu'il faut être connecté sur internet constament pour que notre code fonctionne...
une coupure et c'est le drame. (Je sais ce que c'est, à la 3WA nous avons eu hélas le coup un après-midi... plus de net... fiou...)
